ftrace
An instrumentation tool that automatically replaces floating point operations with user-defined functions
README.md
1 ftrace
2 ======
3 
4 A Pin tool to replace floating-point instructions with user-defined
5 implementations in an IA-32 or x86-64 program.
6 
7 Currently, the only instructions that this tool replaces are:
8 
9  ADDSS
10  SUBSS
11  MULSS
12  DIVSS
13 
14 
15 Setup
16 -----
17 
18 Before running the tool, make sure that you have the Intel PIN library
19 downloaded and that the shell variable PIN_ROOT is set to the path where the PIN
20 library is located. To build the tool, simply run `make` from the main
21 directory. To run the tool, run the following command:
22 
23  pin -t obj-intel64/ftrace.so [-fp_selector_name <fp_selector_name>] -- <your_program>
24 
25 For the IA-32 architecture, use `obj-ia32` instead of `obj-intel64`
26 
27 You can use the `-print_floating_point_operations` flag to print the operands
28 and result of every floating-point arithmetic operation in the instrumented
29 application.
30 
31 You can specify the name of an output file for the operands and result of each
32 floating-point operation by using the `-o` flag. The default name of an output
33 file is `ftrace.out`.
34 
35 Floating-Point Instruction Replacement
36 --------------------------------------
37 
38 Custom floating-point implementation selectors can be written to test the
39 accuracy of different floating-point implementations. Each `FpSelector` selects
40 an `FpImplementation` to replace each SSE floating-point arithmetic instruction.
41 To use an `FpSelector` to instrument an application, the `FpSelector` must first
42 be registered with one of the classes in `src/client_lib/registry`. Once an
43 `FpSelector` has been registered to a name, that name may be used to specify the
44 `FpSelector` to be used to instrument an application with the
45 `-fp_selector_name` flag.
46 
47 See the `tests/` directory for examples of user-defined `FpSelector`s.
48 
49 Testing
50 -------
51 
52 To run the tests for this pintool, run `make test` from the main directory.
